We chose the following idea:
We want to build an AI-powered maintenance request assistant that helps Hausverwaltungen collect, structure, prioritize, and track repair cases across tenants, owners, and service providers.

We also already started a very highlevel idea for the first prototype, focusing on the frontend and maybe one usecase that we can showcase the testers. The idea was to use the time we have left in module 3 to start with the module 4 tasks so that we can get early feedback and implement this feedback in the 2nd prototype.




General things:
THe tenants should be able to create a maintenance requests, with the formatting being helped by the app and verified by the AI

and the owner should be able to see a request and decide if it should be accepted or rejected.

The requests are to be stored in a database most likely
